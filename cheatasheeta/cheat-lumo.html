📟 System Information

10 commands
Kernel, Hostname, CPU, Zeit & mehr — alles, was der Rechner über sich selbst verrät.
uname -aZeigt alle Systeminformationen (Kernel, Architektur, Hostname, etc.)
hostnamectlAktueller Hostname und verwandte Details (wie ein digitaler Ausweis)
lscpuListet die CPU-Architektur auf — Kerne, Taktrate, Flags
timedatectl statusZeigt die Systemzeit, Zeitzone und NTP-Status an
uptimeBetriebsdauer und Systemlast (Load Average) anzeigen
whoamiGibt den aktuellen Benutzernamen aus — "Wer bin ich?"
idZeigt UID, GID und alle Gruppen des aktuellen (oder angegebenen) Users
wListet eingeloggte Benutzer und ihre Aktivitäten auf
lastZeigt die letzten Anmeldungen (aus /var/log/wtmp)
lspciListet alle PCI-Geräte (Grafikkarte, Netzwerk, etc.)

📊 System Monitoring & Management

8 commands
Echtzeit-Überwachung von Prozessen, Speicher, I/O und mehr — der Puls des Systems.
topEchtzeit-Prozessübersicht (interaktiv, sortierbar)
htopInteraktiver Process Viewer (benötigt Installation) — bunter und intuitiver
df -hZeigt Speicherplatz auf Dateisystemen in menschenlesbarer Form (-h)

vmstatZeigt Speicher-, Prozess- und I/O-Statistiken an

sarSystemaktivitätsbericht (aus dem sysstat-Paket) — historische Daten
lsofListet alle offenen Dateien und die dazugehörigen Prozesse auf

⚙️ Prozessverwaltung

10 commands
Prozesse starten, beenden, priorisieren und im Hintergrund laufen lassen.
ps auxZeigt alle laufenden Prozesse mit vielen Details (a = alle, u = user, x = ohne TTY)
kill <PID>Beendet einen Prozess mit der angegebenen Prozess-ID (PID)
killall <name>Beendet alle Prozesse mit einem bestimmten Namen
pkill <pattern>Beendet Prozesse anhand von Namen oder anderen Kriterien
nice -n 19 <cmd>Startet einen Prozess mit niedriger Priorität (nice value)
renice -n 10 -p <PID>Ändert die Priorität eines laufenden Prozesses
nohup <cmd> &Startet einen Prozess, der nach Logout weiterläuft (gegen HUP-Signal geschützt)
<cmd> &Führt den Befehl im Hintergrund aus (Shell kehrt sofort zurück)
jobsListet alle Hintergrund-Jobs der aktuellen Shell auf
fg %1Bringt einen Hintergrund-Job in den Vordergrund (fg = foreground)

📁 Datei- & Verzeichnisverwaltung

12 commands
Die Grundlage jeder CLI-Arbeit: Dateien und Ordner anlegen, verschieben, kopieren, löschen.
lsListet Dateien und Verzeichnisse im aktuellen Ordner auf
ls -laAusführliche Liste (-l) mit allen (-a auch versteckten) Dateien
pwdZeigt den aktuellen Pfad (print working directory)
cd <dir>Wechselt in das angegebene Verzeichnis (cd .. = eine Ebene höher)
mkdir <name>Erstellt ein neues Verzeichnis (make directory)
touch <datei>Erstellt eine leere Datei oder aktualisiert den Zeitstempel
cp <quelle> <ziel>Kopiert Dateien oder Verzeichnisse (cp -r für rekursiv)
mv <quelle> <ziel>Verschiebt oder benennt Dateien/Verzeichnisse um
rm <datei>Löscht eine Datei (Vorsicht! Kein Papierkorb)
rm -rf <dir>Löscht ein Verzeichnis rekursiv und ohne Rückfrage (⚠️ gefährlich!)
ln -s <ziel> <link>Erstellt einen symbolischen Link (Softlink) — wie eine Verknüpfung
file <datei>Erkennt den Dateityp (z.B. ASCII, binary, image)

🔐 Rechte & Eigentum

6 commands
Wer darf was? Lese-, Schreib- und Ausführungsrechte für User, Gruppe und Andere.
chmod u+x <datei>Macht eine Datei für ihren Besitzer ausführbar (+x)
chmod 755 <datei>Setzt rwxr-xr-x — Besitzer: alles, Gruppe/andere: lesen+ausführen
chown <user>:<gruppe> <datei>Ändert Besitzer und Gruppe einer Datei
chgrp <gruppe> <datei>Ändert nur die Gruppe einer Datei
umaskZeigt oder setzt die Standardrechte für neu erstellte Dateien
getfacl <datei>Zeigt erweiterte Zugriffslisten (ACLs) an

🔍 Suchen & Finden

4 commands
Die verlorene Datei? Das versteckte Wort? Mit find und grep wird alles gefunden.
find /pfad -name "*muster*"Sucht rekursiv nach Dateien/Verzeichnissen mit Namensmuster
grep "muster" <datei>Durchsucht eine Datei nach einem Muster (regex-fähig)
grep -r "muster" /pfadDurchsucht rekursiv alle Dateien in einem Verzeichnis nach dem Muster
which <befehl>Zeigt den Pfad einer ausführbaren Datei an

✂️ Textverarbeitung & Filter

12 commands
Die scharfen Werkzeuge der UNIX-Philosophie: grep, awk, sed, cut, sort, uniq & Co.
cat <datei>Gibt den gesamten Dateiinhalt auf der Konsole aus
less <datei>Zeigt eine Datei seitenweise an (mit Vor-/Zurückblättern)
head <datei>Zeigt die ersten 10 Zeilen einer Datei (mit -n beliebig)
tail <datei>Zeigt die letzten 10 Zeilen einer Datei (mit -f live verfolgen)
grep "muster" <datei>Sucht nach Zeilen, die ein Muster enthalten
awk '{print $1}' <datei>Mächtige Textverarbeitung — Spalten extrahieren, Berichte erstellen
sed 's/alt/neu/g' <datei>Stream-Editor — Ersetzungen und Filterungen im Textstrom
cut -d':' -f1 <datei>Extrahiert Spalten (z.B. mit -d Trennzeichen, -f Feld)
sort <datei>Sortiert die Zeilen einer Datei (alphabetisch/numerisch)
uniq <datei>Entfernt doppelte Zeilen (erwartet sortierte Eingabe)
wc <datei>Zählt Zeilen, Wörter und Zeichen (word count)
tr 'a-z' 'A-Z'Ersetzt oder löscht Zeichen (z.B. klein → groß)

📦 Archivierung & Kompression

4 commands
Dateien packen und entpacken — tar ist der Klassiker für .tar.gz, .tar.bz2, .tar.xz.
tar -czvf archiv.tar.gz datei1 datei2Erstellt ein komprimiertes tar.gz-Archiv (c=create, z=gzip, v=verbose, f=file)
tar -xvf archiv.tar.gzEntpackt ein tar.gz-Archiv (x=extract, v=verbose, f=file)
tar -xvf archiv.tar.bz2Entpackt ein tar.bz2-Archiv (bzip2-Kompression)
tar -xvf archiv.tar.xzEntpackt ein tar.xz-Archiv (LZMA-Kompression)

📦 Paketverwaltung — APT

9 commands
Das Herz von Ubuntu: Pakete installieren, aktualisieren, entfernen — mit APT.
sudo apt updateAktualisiert die Paketlisten (macht apt aware von neuen Versionen)
sudo apt upgradeAktualisiert alle installierten Pakete auf die neueste Version
sudo apt install <paket>Installiert ein Paket (oder mehrere, durch Leerzeichen getrennt)
sudo apt install -f --reinstall <paket>Repariert ein beschädigtes Paket (—reinstall)
apt search <begriff>Sucht in den Paketnamen und -beschreibungen nach Begriff
apt-cache policy <paket>Zeigt die verfügbaren Versionen eines Pakets an
sudo apt remove <paket>Entfernt ein installiertes Paket (Konfigurationen bleiben)
sudo apt purge <paket>Entfernt ein Paket samt aller Konfigurationsdateien
add-apt-repository <ppa>Fügt ein PPA-Repository hinzu (Personal Package Archive)

6 commands
Containerisierte Pakete von Canonical — einfache Installation, automatische Updates.
snap find <begriff>Sucht nach Snap-Paketen
sudo snap install <name>Installiert ein Snap-Paket
sudo snap remove <name>Entfernt ein Snap-Paket
sudo snap refreshAktualisiert alle installierten Snaps auf die neueste Version
snap listListet alle installierten Snap-Pakete auf
snap info <name>Zeigt detaillierte Informationen über ein Snap-Paket

👤 Benutzerverwaltung

10 commands
Benutzer anlegen, löschen, Passwörter setzen und Konten sperren.
wZeigt wer gerade eingeloggt ist und was sie tun
sudo adduser <name>Legt einen neuen Benutzer an (interaktiver Assistent)
sudo deluser <name>Löscht einen Benutzer (Optionen: —remove-home)

su <name>Switch User — wechselt zu einem anderen Benutzer

usermod -aG <gruppe> <user>Fügt einen Benutzer zu einer zusätzlichen Gruppe hinzu
sudo userdel <name>Löscht einen Benutzer (low-level, oft mit -r für home)

IP-Adressen, Routing, Sockets, DNS und mehr — die Vernetzung des Systems.

ip -s linkZeigt Netzwerkstatistiken (Pakete, Fehler, etc.)
ss -lListet offene (listening) Sockets auf (modern, ersetzt netstat)
ping <host>Pingt einen Host an und zeigt Latenz/Paketverlust

dig <domain>Erweiterte DNS-Abfragen mit vielen Details
curl <url>Überträgt Daten (HTTP, FTP, etc.) mit vielen Optionen
wget <url>Lädt Dateien aus dem Netz herunter (rekursiv möglich)

rsync -av <quelle> user@host:<ziel>Synchronisiert Dateien effizient über SSH
ssh user@hostStellt eine Remote-Shell über SSH her
nmap <host>Scannt Ports und Dienste auf einem Host (optional installieren)
tcpdumpZeichnet Netzwerkpakete auf (Analyse mit Wireshark möglich)

📐 Netplan-Konfiguration

3 commands
Netplan ist das moderne Netzwerk-Konfigurationstool unter Ubuntu (YAML-basiert).
cat /etc/netplan/*.yamlZeigt die aktuelle Netplan-Konfiguration an
sudo netplan tryTestet eine neue Konfiguration für 120 Sekunden (bei Fehlern wird zurückgesetzt)
sudo netplan applyWendet die aktuelle Netplan-Konfiguration an

🔥 Firewall — UFW

6 commands
Uncomplicated Firewall — einfach zu bedienen, aber mächtig.
sudo ufw statusZeigt den Status der Firewall und die aktiven Regeln
sudo ufw enableAktiviert die Firewall
sudo ufw disableDeaktiviert die Firewall (Vorsicht!)
sudo ufw allow 22Erlaubt Verkehr auf Port 22 (SSH) — auch mit /tcp oder /udp
sudo ufw deny 23Blockiert Verkehr auf Port 23 (Telnet)
sudo ufw delete allow 22Löscht die Regel für Port 22 (allow)
