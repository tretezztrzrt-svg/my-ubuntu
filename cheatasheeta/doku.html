1. .bashrc – Grundkonfiguration

Die Standard-.bashrc für nicht-interaktive Login-Shells. Enthält Prompt-Farben, History-Einstellungen, Auto-Completion und einige Sicherheits-Features.

Ausgewählte Abschnitte

# ~/.bashrc: executed by bash(1) for non-login shells. # If not running interactively, don't do anything case $- in i;; *;; esac # don't put duplicate lines or lines starting with space in history HISTCONTROL=ignoreboth shopt -s histappend HISTSIZE=1000 HISTFILESIZE=2000 shopt -s checkwinsize # make less more friendly for non-text input files [ -x /usr/bin/lesspipe ] && eval "$(SHELL=/bin/sh lesspipe)" # fancy prompt (color if terminal supports) PS1='${debian_chroot:+($debian_chroot)}\[\033[01;32m\]\u@\h\[\033[00m\]:\[\033[01;34m\]\w\[\033[00m\]\$ ' # enable color support for ls alias ls='ls --color=auto' alias grep='grep --color=auto' # load bash_aliases if exists [ -f ~/.bash_aliases ] && . ~/.bash_aliases # programmable completion if ! shopt -oq posix; then [ -f /usr/share/bash-completion/bash_completion ] && . /usr/share/bash-completion/bash_completion fi
⏵ Bemerkung: Die Konfiguration ist solide und weitgehend standardkonform.
✓ History-Einstellungen sinnvoll  |  ✓ Prompt mit Farben  |  ✓ Completion aktiviert
💡 Optimierung: HISTSIZE / HISTFILESIZE auf 5000 erhöhen, wenn du eine längere Befehlshistorie wünschst.

2. Funktionen (vorher2.txt)

Sechs selbstdefinierte Funktionen für Suche, Archivierung und Verzeichniswechsel. Einige sind mit buggy markiert – hier gibts Verbesserungen.

2.1 search() buggy? nochmals

# Durchsucht Dateien rekursiv nach einem Suchbegriff # Nutzung: search "suchbegriff" function search() { if [ -z "$1" ]; then echo "Verwendung: search <suchbegriff>" return 1 fi egrep -roi "$1" . | sort | uniq }
⚠ Probleme:
  • egrep ist veraltet – verwende grep -E.
  • Keine Möglichkeit, das Suchverzeichnis anzugeben (immer .).
  • Binärdateien werden nicht ausgeschlossen (-I fehlt).
  • sort | uniq kann durch sort -u ersetzt werden.
✓ Verbesserte Version:
search() {
  local dir="${2:-.}"
  grep -rEoiI "$1" "$dir" 2>/dev/null | sort -u
}
→ Nutzung: search "muster" [pfad]

2.2 whereis() buggy

# Findet Dateien im aktuellen Verzeichnis, die mit dem Suchbegriff beginnen function whereis() { if [ -z "$1" ]; then echo "Verwendung: whereis <dateiname>" return 1 fi find . -type f -name "$1*" }
⚠ Probleme:
  • Namenskonflikt mit dem systemeigenen whereis (zeigt Speicherort von Binaries).
  • Sucht nur im aktuellen Verzeichnis – kein rekursiver Pfad.
  • Keine Groß-/Kleinschreibung ignorieren (-iname).
✓ Verbesserte Version (umbenannt):
ff() { # "find file"
  local dir="${2:-.}"
  find "$dir" -type f -iname "$1*" 2>/dev/null
}
→ Nutzung: ff "datei" [pfad]   (sucht rekursiv)

2.3 mkdircd() ubuntu-proofed

# Erstellt einen Ordner und wechselt sofort hinein function mkdircd() { if [ -z "$1" ]; then echo "Verwendung: mkcd <ordnername>" return 1 fi mkdir -p "$1" && cd "$1" }
⏵ Bewertung: Funktioniert einwandfrei.
✓ -p erstellt fehlende Zwischenverzeichnisse.
✓ Fehlerbehandlung vorhanden.
💡 Du könntest zusätzlich ls -la nach dem Wechsel automatisch ausführen – dann siehst du sofort den Inhalt.

2.4 entpacken() hat geklappt?

# Entpackt .7z Archive function entpacken() { if [ $# -ne 1 ]; then echo "Error: No file specified." return 1 fi local archiv="$1" if [ ! -f "$archiv" ]; then echo "'$archiv' is not a valid file" return 1 fi case "$archiv" in *.7z|*.7Z) 7z x -y "$archiv" ;; *) echo "'$archiv' ist kein .7z Archiv"; return 1 ;; esac }
⚠ Problem: Unterstützt nur 7z – sehr eingeschränkt. 💡 Ein universelles entpacken sollte .tar.gz, .zip, .rar, .bz2 usw. erkennen.
✓ Erweiterte Version (Multiformat):
entpacken() {
  case "$1" in
    *.tar.gz|*.tgz) tar -xzvf "$1" ;;
    *.tar.bz2) tar -xjvf "$1" ;;
    *.tar.xz) tar -xJvf "$1" ;;
    *.zip) unzip "$1" ;;
    *.rar) unrar x "$1" ;;
    *.7z) 7z x "$1" ;;
    *) echo "Unbekanntes Format: $1" ;;
  esac
}
→ Voraussetzung: unrar, unzip, 7z installiert.

2.5 packen() hat geklappt

# Packt Datei/Ordner in .7z (normale Kompression) function packen() { if [ $# -lt 2 ]; then echo "Verwendung: packen <datei_oder_ordner> [zielname]" return 1 fi local quelle="$1" local ziel="${2:-$(basename "$quelle").7z}" case "$ziel" in *.7z;; *) ziel="${ziel}.7z" ;; esac if [ ! -e "$quelle" ]; then echo "Fehler: Quelle existiert nicht"; return 1; fi if [ -e "$ziel" ]; then echo "Fehler: $ziel existiert bereits!"; return 1; fi 7z a -t7z "$ziel" "$quelle" }
⏵ Bewertung: Solide, mit guter Fehlerprüfung.
✓ Quellprüfung & Ziel-Kollisionscheck.
✓ Automatische .7z-Ergänzung.
💡 Du könntest eine Fortschrittsanzeige mit -bsp1 aktivieren.

2.6 packen-hard() buggy? nochmals

# Packt mit maximaler Kompression (-mx=9) function packen-hard() { # ... gleiche Parameter wie packen() ... 7z a -t7z -mx=9 "$ziel" "$quelle" }
⚠ Problem: Code-Duplizierung – packen() und packen-hard() sind fast identisch, nur der -mx-Parameter unterscheidet sich.
💡 Besser: eine Funktion mit optionalem Kompressionslevel.
✓ Vereinheitlichte Version:
packen() {
  # ... Parameterprüfung wie oben ...
  local level="${3:-5}" # Standard: 5 (ausgewogen)
  7z a -t7z -mx="$level" "$ziel" "$quelle"
}
→ Nutzung: packen quelle [ziel] [level]   (level 0–9)
→ packen-hard wird dann zu packen quelle ziel 9

3. Aliases (vorher3.txt)

Über 50 Aliases für Navigation, Systeminfo, Paketverwaltung, Dateioperationen und mehr. Einige sind doppelt, andere überschreiben systemeigene Befehle – hier eine strukturierte Übersicht mit Verbesserungsvorschlägen.

3.1 System & Shortcuts

alias c='clear' # Terminal leeren alias q='exit' # Terminal beenden alias h='history' # Historie anzeigen alias x='chmod +x' # Skript ausführbar machen alias r='shutdown -r now' # System sofort neustarten
⏵ Bemerkung: Kurze, einprägsame Aliases – gut!
💡 r für Reboot ist praktisch, aber gefährlich: verwende shutdown -r +1 für einen verzögerten Neustart, um versehentliche Ausführungen zu vermeiden.

3.2 Navigation & Verzeichnisse

alias alex='cd /home/alex/' alias home='cd /home/alex/' alias vera='cd /media/veracrypt1/' alias var='cd /var/' alias logs='cd /var/log/' alias doc='cd ~/Documents/' alias down='cd ~/Downloads/' alias etc='cd /etc/' alias ..='cd .. && ls -la' alias ...='cd ../../ && ls -la' alias ....='cd ../../../ && ls -la' alias cd..='cd .. && ls -la'
✓ Verbesserung:
✓ home und alex sind doppelt – behalte nur home.
✓ Für die cd..-Aliases gibt es eine elegantere Lösung:
alias ..='cd .. && ls -la' alias ...='cd ../.. && ls -la' alias ....='cd ../../.. && ls -la'
(die Punkte zählen die Ebenen – cd.. ist redundant)

3.3 Hardware & Systeminfo

alias osinfo='lsb_release -a || cat /etc/os-release' alias cpuinfo='lscpu' alias hardware='lshw -short' alias systemstatus='top -b -n 1 | head -n 20' alias uptime_p='uptime -p' alias free='free -h --si' alias raminfo='free -h' alias platz='df -hPT | column -t' alias df='df -h' alias space='df -h' alias diskinfo='lsblk' alias path='echo -e ${PATH//:/\n}'
⚠ Konflikte & Redundanzen:
  • free überschreibt den systemeigenen free – besser alias freeh='free -h'.
  • df überschreibt df – verwende alias dfh='df -h'.
  • platz, space, df sind fast identisch – behalte nur platz (mit column -t).
  • raminfo und free sind redundant.
✓ Säuberung:
alias freeh='free -h' # RAM in lesbar
alias dfh='df -hPT | column -t' # Plattenplatz
→ So behältst du die Originalbefehle und hast trotzdem praktische Shortcuts.

3.4 Paketmanagement

alias install='sudo apt install ' alias update='sudo apt update && sudo apt full-upgrade -y && sudo snap refresh && sudo flatpak update -y' alias upgrade="sudo apt update && sudo apt upgrade -y" alias autoclean='sudo apt autoremove && sudo apt autoclean' alias uninstall='sudo apt remove' alias cleanup='sudo apt autoremove -y && sudo apt clean'
⏵ Bewertung: Sehr gute All-in-One-Updates!
✓ update aktualisiert apt, snap und flatpak – praktisch.
💡 Achte darauf, dass full-upgrade auch Kernel-Updates einspielt – das ist meist erwünscht.
💡 install mit einem Leerzeichen am Ende ist gut, aber du könntest install als Funktion definieren, die sudo apt install mit Fehlerbehandlung erweitert.

3.5 Dateioperationen & Sicherheit

alias cp='cp -i' alias mv='mv -i' alias rm='rm -i' alias ls='ls -la --color=auto' alias l='ls -la --color=auto' alias sl='ls -la --color=auto' alias ll='ls -alF' alias la='ls -A' alias dir='dir --color=auto' alias vdir='vdir --color=auto'
⚠ Probleme:
  • ls überschreibt den systemeigenen ls mit -la – das ist gewöhnungsbedürftig, da du immer alle versteckten Dateien siehst.
  • l, sl und ll sind teilweise redundant.
  • rm -i ist eine gute Sicherheitsmaßnahme – beibehalten!
✓ Verbesserung:
alias l='ls -la --color=auto' # ausführlich
alias ls='ls --color=auto' # Originalverhalten + Farbe
alias ll='ls -alF --color=auto' # klassisch ll
→ So bleibt ls Standard, l zeigt alles.

3.6 Ordner-Größen & Speicher

# DAS IST ALLES BUGGY alias ordner='du -h' alias sortiert='du -sh * | sort -h' alias bigfiles='du -ah . | sort -rh | head -n 10' alias lsize='du -sh * 2>/dev/null | sort -hr | head -n 10'
⚠ Die "buggy"-Markierung ist berechtigt:
  • sortiert sortiert nicht numerisch – sort -h ist korrekt, aber du -sh * zeigt nur die oberste Ebene an.
  • bigfiles sucht im gesamten aktuellen Verzeichnis – das kann sehr langsam werden bei großen Bäumen.
  • lsize unterdrückt Fehler (2>/dev/null), aber sort -hr erfordert GNU sort (bei macOS anders).
✓ Verbesserte Versionen:
alias topdirs='du -sh * 2>/dev/null | sort -hr | head -10' # Top 10 Ordner
alias topfiles='find . -type f -exec du -h {} + 2>/dev/null | sort -hr | head -20' # Top 20 Dateien
→ topdirs ist schnell, topfiles durchsucht rekursiv.

3.7 Korrektur-Aliases Klassiker

alias bitte='sudo $(history -p !!)' alias fuck='sudo $(history -p !!)' alias please='sudo $(history -p !!)' alias pls='sudo $(fc -ln -1)'
⏵ Bewertung: Ein Muss für jeden Admin!
✓ Funktioniert zuverlässig.
💡 Du hast vier Aliases für die gleiche Aktion – behalte fuck und pls, die anderen sind redundant.

3.8 Logs & Textsuche

alias journal='sudo journalctl -xe' #buggy# alias fh='find . -name' alias ftext='grep -rnw . -e' alias tailf='tail -f' alias tailn='tail -n 50' alias cat='less' alias more='less'
⚠ Bemerkungen:
  • journal ist nicht "buggy", es fehlt nur die Fehlerbehandlung – aber mit sudo funktioniert es. Evtl. sudo -E für Umgebungsvariablen.
  • cat=less ist ein gewagter Ersatz – cat wird oft in Skripten verwendet und sollte nicht überschrieben werden. Besser alias cless='less'.
  • fh ist redundant zur ff()-Funktion (s.o.) – verwende ff.

4. Zusammenfassung & Empfehlungen

✔️ Beibehalten & gut

  • • mkdircd() – perfekt
  • • packen() – solide
  • • cp -i, mv -i, rm -i
  • • fuck/pls – Klassiker
  • • update (All-in-One)
  • • pw, profile_me

🔧 Überarbeiten

  • • search() → grep -E + Pfad
  • • whereis() → umbenennen zu ff()
  • • entpacken() → Multiformat
  • • packen-hard() → in packen integrieren
  • • df, free nicht überschreiben
  • • ls nicht mit -la überschreiben

🗑️ Redundant / Entfernen

  • • alex (doppelt zu home)
  • • cd.. (redundant zu ..)
  • • bitte, please (nur fuck behalten)
  • • space (doppelt zu platz)
  • • raminfo (doppelt zu free)
  • • sl (typo-Korrektur, aber l reicht)
  • • cat=less – gefährlich, besser weglassen

📌 Allgemeine Empfehlungen

  • Aliases in ~/.bash_aliases auslagern – hält .bashrc sauber.
  • Funktionen in ~/.bash_functions auslagern und in .bashrc sourcen.
  • Dokumentation – füge für jede Funktion/Alias eine kurze Hilfe hinzu (z.B. mit comment).
  • Teste Änderungen mit bash -n ~/.bashrc und source ~/.bashrc.
  • Backup – sichere deine aktuelle Konfiguration, bevor du große Änderungen vornimmst.
⏵ Fazit: Du hast eine sehr umfangreiche und durchdachte Bash-Konfiguration. Mit einigen Anpassungen wird sie noch robuster, schneller und besser wartbar. Die markierten "buggy"-Stellen sind meist nur Kleinigkeiten – die vorgeschlagenen Verbesserungen machen dein Setup fit für den täglichen Einsatz.
